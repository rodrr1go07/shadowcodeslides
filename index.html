Identificação do problema

O Problema

🌐

Acesso limitado

Jogos de tabuleiro online geralmente exigem instalação, cadastro ou são pagos, dificultando o acesso rápido pelo navegador.

🇧🇷

Barreira do idioma

Poucas plataformas oferecem experiência nativa em português — o que afasta parte significativa do público brasileiro.

👥

Falta de opção gratuita

Grupos de amigos precisam de uma plataforma gratuita e de fácil acesso para jogar juntos sem barreiras técnicas.

04 / 12
Principais Funcionalidades

Como o jogo funciona?

🏠

Salas com código único

Admin cria uma sala e compartilha o código gerado automaticamente. Outros jogadores entram com o código, sem cadastro.

🎭

Papéis: Mestre & Agentes

O Mestre dá dicas de uma palavra com um número. Os Agentes votam nas 25 cartas tentando revelar as corretas.

🃏

Tabuleiro 5×5

25 cartas divididas em: 10 corretas (douradas), 12 neutras e 3 assassinas. Revelar uma assassina encerra o jogo.

🕵️

Impostor secreto

Um agente é sorteado como impostor — ele vê as cartas assassinas e tenta sabotar a equipe discretamente.

🗳️

Sistema de votação

3 votos são necessários para revelar uma carta, incentivando debate e consenso entre os agentes.

⏱️

Timer de rodada

O admin pode configurar um limite de tempo por rodada, aumentando a tensão e o ritmo da partida.

Stack técnica

Tecnologias utilizadas

⚛️ Frontend

  • React — biblioteca de UI declarativa
  • Vite — bundler ultrarrápido para desenvolvimento
  • React Router DOM — roteamento client-side
  • Socket.IO — comunicação em tempo real
  • CSS3 — estilização responsiva customizada

🖥️ Backend & Banco

  • Node.js + Express — servidor HTTP e API REST
  • Socket.IO — servidor de WebSocket para eventos
  • SQLite — banco leve com WAL para escrita concorrente
  • dotenv + CORS — configuração de ambiente e segurança
React Vite Node.js Express Socket.IO SQLite React Router CSS3

🔌 Eventos Socket (Lobby)

  • lobby_create → gera código da sala
  • lobby_join_code → jogador entra com código
  • lobby_choose_role → seleção de papel
  • lobby_start → admin inicia a partida

🎮 Eventos Socket (Jogo)

  • vote_card → agente vota em uma carta
  • card_revealed → carta virada (broadcast)
  • clue_submitted → mestre envia dica
  • game_finished → vencedor determinado

Obstáculos & Soluções

Desafios enfrentados

  • ⚡

    Sincronização em tempo real entre múltiplos jogadores

    Garantir que votos, cartas reveladas e dicas chegassem a todos simultaneamente sem inconsistências. → Resolvido com Socket.IO e broadcast de eventos por sala.

  • 🗃️

    Gerenciamento de estado entre banco e memória

    Dados de lobby precisavam de resposta imediata; dados de jogo precisavam de persistência. → Separação: lobbies em memória (Map), partidas no SQLite com WAL.

  • 🕵️

    Mecânica do impostor sem revelar identidade

    O impostor precisa ver as cartas assassinas sem que os outros saibam quem ele é. → Estado do impostor enviado apenas para o socket individual do jogador sorteado.

  • 📱

    Responsividade do tabuleiro em telas pequenas

    O grid 5×5 com sidebar colapsava em dispositivos móveis. → Drawer lateral com toggle e media queries para adaptar o layout ao mobile.
